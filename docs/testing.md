# 测试与验收

## 本地回归

后端/静态回归在 Docker 镜像内运行，避免本机 Python 缺少 Flask 等依赖：

```bash
docker run --rm -v "$PWD":/work -w /work long-novel-gpt:local \
  python -m unittest tests.test_v2_workflow tests.test_frontend_static tests.test_security_regressions
```

前端构建：

```bash
cd frontend2
npm run build
```

## 关键用例

- TXT/DOCX 导入：中文不乱码，10w 字以内接受，100001 字拒绝。
- ZIP 导入：只作为内部评测集，不进入小说列表。
- 自动拆分：无标题长文按 2200/3000 字拆分，拼回正文去空白后等于原文。
- 洗稿：默认 `auto`；首轮无严重质量问题直接保存，结构相似、重合过高、表层换皮不足、内部重复、视角漂移、流水账、开头钩子不足等严重问题才同模型重洗 1 次；轻微篇幅、节奏轻微拖沓、开头修饰过密、AI 套话轻微等只标记复查；截断、空正文、保存失败或取消后的结果不保存为成稿。
- 转剧本：输入必须是洗稿结果，不重新换皮，不覆盖洗稿字段。
- 导出：原稿、洗稿、剧本三类导出顺序一致；洗稿/剧本未全部生成时阻止导出。

## 浏览器验收

在工作台页面依次验证：

1. 导入几千字样本，确认自动拆分或短文本快路径正常。
2. 点击整本洗稿，确认创建后台任务、章节显示排队中/生成中/质量复查/已完成等状态，刷新页面可恢复进度，任务完成后无客户可见内部提示词和指标噪声。
3. 点击转剧本，确认剧本生成后可切换查看。
4. 尝试导出洗稿/剧本，确认未生成章节会被拦截，完整生成后可下载。
5. 在桌面、平板宽度、手机宽度检查导入弹窗、章节列表、结果区和导出菜单不重叠。

## 实跑记录

每次用真实模型跑客户资料时，把摘要保存到 `artifacts/` 或 `tmp/`，至少包含：

- 样本文字数、拆分段数、模型名、并发数。
- 总耗时、每段成功/失败状态。
- 每段 `quality.delivery_status`、`overlap4`、`structure_similarity`、`length_ratio`；1800 字以上长章节允许到约 130%，中短章节仍按 118% 上限。
- 是否出现 `self_distinct4` 偏低、`repeated_phrases` 或“内部重复”问题。
- 是否出现“开头钩子不足”“节奏拖沓”，并人工抽看前 200 字是否符合短剧短快爽、带引号直接对白优先、硬冲突 80 字内落地。
- 是否完成转剧本和导出。
